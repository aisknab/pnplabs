Run command

Run the source/checker verifier

git clone https://github.com/aisknab/pnp.git
cd pnp
npm ci
npm run pnp:verify

Attach the final verifier JSON or a CI log showing the final verdict. The canonical local output is artifacts/pnp-verify-all/latest-verdict.json.

Focused activation checks

Include activated proof-script evidence when possible

npm run proof:activated-pnp-status
npm run proof:public-theorem-activation
npm run proof:unrestricted-final-soundness-release
npm run proof:uniform-complexity-conclusion

These focused outputs make it easier to bind a public run to the activated status coordinate and the theorem-emission gate.

What a run report means

A run report means someone reports running the verifier and reproducing the activated checker-trust status. It is not an external-consensus claim or peer-review acceptance.

Submission checklist

What to include

  • Runner name or handle to list publicly.
  • UTC date and exact aisknab/pnp commit or ref tested.
  • OS, CPU architecture, Node version, npm version, and container or VM details if used.
  • Commands run and any deviations from the canonical command.
  • Final verifier JSON, logs, CI link, or checksums sufficient for reproduction.
  • Activated status coordinate PNP-ACTIVATED-STATUS-2026-07-05-01 and public theorem activation coordinate PNP-PUBLIC-THEOREM-ACTIVATION-2026-07-05-01.
  • SHA-256 digest or artifact link for public/pnp-status.json or status/ACTIVATED_PNP_STATUS.json, where available.
  • Focused proof-script outputs for proof:activated-pnp-status, proof:public-theorem-activation, proof:unrestricted-final-soundness-release, and proof:uniform-complexity-conclusion, where available.
