Normalized reproducibility comparison

Compare activated verifier runs by normalized digest.

Use normalized digest fields to compare activated checker-trust verifier runs across local machines and CI systems without depending on raw log formatting.

Open comparison JSONRun registry
Comparison command

Compare two run records

npm run pnp:compare-runs -- --json <left-run.json> <right-run.json>

The comparison utility normalizes missing digest fields on the fly, then compares the activated verdict, activated status, focused proof-script outputs, raw-log evidence digests, and full normalized run record digests.

Default required digests

What must match by default

verdictNormalizedSha256

Activated theorem verdict fields match: publicTheoremEmissionAllowed = true, publicTheoremStatement = "P = NP", and remainingBlockers = [].

activatedStatusNormalizedSha256

Activated status coordinates match, including PNP-ACTIVATED-STATUS-2026-07-05-01 and PNP-PUBLIC-THEOREM-ACTIVATION-2026-07-05-01.

proofScriptOutputsNormalizedSha256

Focused proof-script accept outputs match after normalization.

Optional full evidence match

artifactsOrLogsNormalizedSha256 and runRecordNormalizedSha256 help compare raw log evidence and full metadata, but runner names and CI links may intentionally differ.

Digest fields

All normalized digest fields

runRecordNormalizedSha256
verdictNormalizedSha256
activatedStatusNormalizedSha256
proofScriptOutputsNormalizedSha256
artifactsOrLogsNormalizedSha256

The digest policy is PNPActivatedRunDigestNormalization0. It normalizes timestamps, ANSI escapes, CRLF/LF line endings, blank lines, and trailing whitespace before digesting log-like fields.

A digest match is reproducibility metadata for activated checker-trust verifier runs. It is not an external-consensus claim or peer-review acceptance. A digest mismatch is a triage signal; it does not by itself refute the source/checker proof stack.
