Independent verification

Verification protocol for the sealed proof-report release.

Independent review can proceed in two complementary modes: verification of the committed artefact bundle, and regeneration of the compact and full proof-report records from the recorded source/checker revision.

Mode A

Artefact verification

Check out the sealed artefact release, verify durable files against SHA256SUMS while excluding any SHA256SUMS self-entry, then inspect the compact and full proof-report records.

Mode B

Regeneration verification

Check out the recorded source/checker revision, run validation, regenerate compact and full proof reports, and confirm the accepted theorem fields and package/replay/certificate linkage.

Success criteria

Acceptance criteria.

Accepted final report

CheckFinalPNPProofReport0 accepts with theorem statement P = NP and antecedent CheckPCCPackexp(GeneratePCCPack()) = accept.

Accepted linkage

The package checker, acceptance run, replay, final verdict, final certificate, release audit, and release gate all accept.

Canonical-byte discipline

Hash lookups are followed by full key or canonical-byte comparison, and generator drift is rejected by canonical-byte comparison.

External audit checklist

Baseline checks for independent review.

Inspect no-hidden-minimization

Confirm executable occurrences of hidden minimization symbols are rejected after macro and alias expansion.

Audit the mode firewall

Verify quotient equalities are not consumed as constructive full-mode replacements.

Check the final framework match

Confirm Package O and Package G use the same NAND syntax, output convention, charge convention, minimum-size notion, and slack definition.
