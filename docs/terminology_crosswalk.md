# Terminology Crosswalk

This crosswalk translates repository and report terminology into more standard complexity-theory, proof-engineering, and formal-methods vocabulary. It is written for skeptical review: a row states only what this checkout, the bundled report, or the source/checker audit map supports. Anything not directly executable from this checkout is marked as a source/checker audit target, `needs author confirmation`, or as requiring independent mathematical review.

Uncertainty labels mean:

- **Confirmed locally**: the public checkout contains the enforcing code or file and a local command can check it.
- **Report-stated**: the bundled report names the definition or checker claim, but this checkout cannot execute the theorem-level checker.
- **Needs author confirmation**: the exact schema, generated artefact, proof obligation, or enforcing source/checker path is not present in this checkout. The term should not be treated as fully defined until the named source/checker revision, generated artefacts, or author-supplied definition is inspected.
- **Independent review required**: even if a source/checker run reproduces the reported acceptance status, the mathematical implication or checker soundness still requires expert review.

Checker paths have two different statuses:

- **Local fixture checker**: code in [tools/reviewer-fixture-checker.mjs](../tools/reviewer-fixture-checker.mjs). It checks only tiny JSON fixtures in [examples/minimal/](../examples/minimal/). It does not check the theorem, the real generated package, or the source/checker implementation named by the report.
- **Report-stated checker**: a checker named in [downloads/canonical_proof_report.tex](../downloads/canonical_proof_report.tex). This checkout contains the report, not the full source/checker repository, so these paths are audit targets rather than independently executable evidence here.
- **Source/checker audit target**: a path in `aisknab/pnp@final-pnp-proof-report-hardened-7072f8d:<path>`, mapped in [source_checker_map.md](source_checker_map.md). Running the source/checker tests checks that package according to its implementation; it is not external mathematical acceptance.
