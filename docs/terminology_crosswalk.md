# Terminology Crosswalk

This crosswalk translates repository and report terminology into more standard complexity-theory, proof-engineering, and formal-methods vocabulary. It is written for skeptical review: a row states only what this checkout or the bundled report supports. Anything not directly confirmable from this checkout is marked `needs author confirmation`.

Checker paths have two different statuses:

- **Local fixture checker**: code in [tools/reviewer-fixture-checker.mjs](../tools/reviewer-fixture-checker.mjs). It checks only tiny JSON fixtures in [examples/minimal/](../examples/minimal/). It does not check the theorem, the real generated package, or the source/checker implementation named by the report.
- **Report-stated checker**: a checker named in [downloads/canonical_proof_report.tex](../downloads/canonical_proof_report.tex). This checkout contains the report, not the full source/checker repository, so these paths are audit targets rather than independently executable evidence here.

This checkout contains prose descriptions of those objects but not the source/checker artefacts needed to verify exact schemas, proof rules, or enforcement paths. Treat their detailed definitions as `needs author confirmation` unless an audit has the named source/checker revision and generated artefacts in hand.
