This crosswalk translates current formal-reconstruction terms and historical assertion-checker terms
into more standard complexity-theory, proof-engineering, and formal-methods vocabulary. The current
canonical download is the twenty-seven-page inventory-derived report. Every bare `Report` citation in the
historical tables below refers only to the 56-page manuscript at source tag
`final-pnp-proof-report-hardened-7072f8d`, commit
`7072f8d0bda6d44d240f9bb3fad624fd357e1278`; it never refers to the current canonical aliases.
Anything not directly executable from this checkout is marked with an evidence-status label rather
than inferred.

Evidence-status labels mean:

- **Confirmed in public checkout**: this checkout contains the enforcing public file or local fixture code and a local command can check it.
- **Defined in pinned source; independent source audit required**: the term has a pinned source/checker coordinate under `sourceRef`; the implementation still needs independent inspection.
- **Present only in the historical report; formal source evidence required**: the pinned 7072f8d manuscript defines or uses the term, but that prose is not current formal evidence.
- **Generated artefact required**: the term depends on generated proof-report artefacts pinned under `artifactRef`.
- **Missing or ambiguous; author clarification requested**: pinned source, release docs, and generated artefacts do not resolve the definition. Author clarification can resolve ambiguity but is not verification evidence.
- **Independent mathematical review required**: the term or implication remains a mathematical-review target even if implementation coordinates are valid.

Checker paths have two different statuses:

- **Local fixture checker**: code in [tools/reviewer-fixture-checker.mjs](../tools/reviewer-fixture-checker.mjs). It checks only tiny JSON fixtures in [examples/minimal/](../examples/minimal/). It does not check the theorem, the real generated package, or the source/checker implementation named by the report.
- **Historical report-stated checker**: a checker named by the pinned 7072f8d manuscript. The
  current canonical report does not adopt those checker assertions. The mapped source paths are
  historical audit targets rather than current theorem evidence.
- **Pinned cross-repo target**: a path assigned to `sourceRef`, `docsRef`, or `artifactRef` in [source_checker_map.md](source_checker_map.md) and [audit_targets.json](audit_targets.json). Running source/checker tests checks the package according to its implementation; it is not external mathematical acceptance.
