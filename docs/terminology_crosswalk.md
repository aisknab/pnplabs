# Terminology Crosswalk

This table is written for reviewers who prefer conventional complexity-theory and proof-engineering vocabulary before internal names. "Checker module/function" names the checker reported by the paper when available. When this checkout only has an educational fixture check, that is stated explicitly.

| Internal term | First defined in file/section | Formal definition | Closest standard concept, if any | Why a new term is needed | What would break if this definition were wrong | Minimal example | Checker module/function that enforces it | Related theorem/lemma |

| locked NAND | `downloads/canonical_proof_report.tex`, "Locked NAND and final integration artefacts"; Appendix A | A constrained multi-output NAND direct-wire construction with lock/private inputs, slot disjointness, macro truth-table checks, threshold certificate, and residual slack bound. | Restricted circuit encoding or gadget reduction. | The construction adds lock/slot conditions beyond ordinary NAND circuits. | SAT reduction, threshold theorem, and residual slack bound could be false. | `examples/minimal/pass-basic-all-invariants.json` uses NAND truth table `1110` with a lock input. | Report: `CheckGPack`, `CheckFinalInt_exp`; local fixture: `checkLockedNand`. | "Locked NAND and final integration"; Package G rows. |
| residual slack | `downloads/canonical_proof_report.tex`, "Central scale correction" | `Lambda(C) = |C| - mu(C)`, where `mu(C)` is minimum equivalent circuit size in the chosen framework. | Gap between current circuit size and optimum. | The descent measure is not root size; it tracks distance from optimum. | A false slack value could hide exponential minimization or invalid descent. | Passing fixture has size `6`, minimum `2`, slack `4`. | Report: Package O/G slack checks; local fixture: `checkResidualSlack`. | Residual-band minimization theorem; locked NAND residual slack at most four. |
| PCCPack | `downloads/canonical_proof_report.tex`, "Executive status and claim boundary" | A finite proof-carrying package emitted by `GeneratePCCPack()` and accepted by `CheckPCCPackexp0`. | Proof-carrying code/package, certificate bundle. | The proof is represented as generated finite data, not just prose. | The antecedent `CheckPCCPackexp(GeneratePCCPack()) = accept` would not be meaningful. | Minimal fixture has `pccpack.version`, rows, import ledger, duplicate-row ledger. | Report: `CheckPCCPackexp0`; local fixture: `checkPccpack`. | Generated package final acceptance; package sufficiency. |

| mode firewall | `downloads/canonical_proof_report.tex`, "Mode firewall and transfer identity" | A quotient equality is not a full equality; constructive full-mode use requires a full lift and discharged lost-bit/finite-kernel obligations. | Type discipline or abstraction boundary between quotient and full semantics. | Projection-based comparisons are useful but unsafe as replacements. | Quotient witnesses could be used to prove invalid full-mode rewrites. | `fail-mode-firewall-violation.json` consumes quotient equality as a full replacement. | Report: `CheckModeUse0`, Package E obligation checks; local fixture: `checkModeFirewall`. | Projection monotonicity; transfer identity; VerifyDW soundness. |

| certificate | Multiple sections; e.g. "Package E: direct-wire verifier" | Finite data whose fields justify a checker conclusion, such as `(C,U,R,Pi)` for a constructive gain. | Certificate or proof witness. | The system separates claims from checkable evidence. | Checker acceptance could become assertion-based rather than evidence-based. | Minimal fixture certificate records minimum size, residual slack, parser digest, ZeroSlack status. | Report: checker-specific certificate validators; local fixture checks certificate fields. | VerifyDW soundness; final certificate theorem. |
| seal | `downloads/canonical_proof_report.tex`, "Final proof-report release seal"; `downloads/release-seal.json` | SHA-256 manifest binding named files or canonical objects to digests. | Checksum manifest or release attestation. | Reviewers need stable artefact identity across publication channels. | Reviewers could audit different bytes than the reported artefacts. | `seal.payloadText` plus SHA-256 in minimal fixtures. | Report: release audit/gate; local: `tools/verify-release-seal.mjs`, `checkSeal`. | Final proof-report release seal. |
| checker acceptance | `downloads/canonical_proof_report.tex`, "Hardened checker suite" | Deterministic `accept` record with normal form, digest, and ledger, or deterministic `reject` record with coordinate, path, witness, and digest; no `unknown`. | Decidable verifier result. | The proof boundary is executable acceptance, not informal confidence. | Ambiguous or partial checker output would weaken replay and final theorem extraction. | `ACCEPT pass-basic-all-invariants` from the fixture checker. | Report: all `Check*` functions; local: `tools/reviewer-fixture-checker.mjs`. | Checker hardening; final proof report theorem. |
| SAT-to-minimization route | `downloads/canonical_proof_report.tex`, "Main message" | Reduction from SAT to exact residual-band minimization of a locked NAND word, followed by a polynomial minimization result. | Karp reduction plus optimization problem. | The route is not a standard textbook reduction; it needs named intermediate objects. | Failure breaks the implication from package acceptance to `SAT in P`. | Schematic example: map a one-clause SAT instance to a NAND-word gadget, then decide SAT by checking the gadget's certified minimization threshold. | minimization trace | Report uses trace/replay language across normalization and acceptance sections | A recorded sequence of normalizations, replacements, routes, or replay steps that justifies a minimization-related conclusion without executing hidden exact search. | Proof trace or replay certificate. | Reviewers must distinguish checked traces from untrusted search. | Hidden search could be disguised as a trace. | The fixture's executable row is intentionally non-minimizing; failing hidden-min example rejects. | Report: replay checker, row-family checkers, no-hidden-min scan. | Final acceptance run; traceable normalization. |
| witness | Many sections, e.g. "Mode firewall", "Residual witness normal form" | Finite object establishing a local claim or failure, such as a full-mode cancellation witness, Hall witness, or positive residual witness. | Mathematical witness. | Different branches require different concrete evidence. | A missing or wrong witness can turn an implication into an assertion. | `positiveResidualWitness` in fixture ZeroSlack field. | Report: package-specific witness validators; local ZeroSlack and mode checks. | VerifyDW; residual witness normal form; BN5 localization. |
| invariant | Theorem environment and package ledgers throughout report | A property preserved by proof steps or required by a checker ledger. | Inductive invariant. | The proof has many mode, charge, route, and import conditions. | A broken invariant can invalidate downstream package conclusions. | Fixture invariants include acyclic imports, no duplicate row keys, zero open obligations. | Report: row/global ledgers; local fixture checker. | Checker hardening; global proof DAG. |
| firewall | Report uses mode, quotient, constructive, and global firewalls | A boundary preventing facts from one semantic layer or package from being consumed in another without explicit checks. | Type abstraction boundary, information-flow control. | The proof uses projections and package separation that are unsafe if freely mixed. | Circularity, invalid replacement, or cross-package assumption leakage. | Mode-firewall failing fixture. | Report: `CheckModeUse0`, global firewalls; local `checkModeFirewall`. | Mode firewall and transfer identity; global proof DAG. |
| residual | Report uses residual slack, residual witnesses, residual keys, residual mass | A remaining gap, witness, or signed ledger entry after cancellations or comparisons. | Residual/gap term in optimization or accounting. | Several proof packages track what remains after local cancellations. | A sign or accounting error could create false descent or false closure. | Residual slack field in minimal fixture. | Report: residual-band package checks; local `checkResidualSlack`. | BN4 residual ledger; residual-band theorem. |
| direct-wire word | `downloads/canonical_proof_report.tex`, "Direct-wire words" | Sequence of NAND gates where each source is a boundary coordinate, constant, or earlier gate output, with ordered output tuple. | Boolean circuit DAG in topological order. | The proof needs a canonical circuit representation with carriers and profiles. | Parser, size, and equivalence claims could refer to different objects. | Minimal fixture source text describes one NAND truth table, not a full direct-wire word. | Report: `VerifyDW`, parser/canonical encoder. | Base direct-wire framework; VerifyDW soundness. |
| carrier | `downloads/canonical_proof_report.tex`, "Direct-wire words" | Tuple containing boundary, interface/output tuple, and profile metadata for constants, slots, origins, kernels, obligations, directions, saturation, budget, and charge. | Typed context or interface. | Circuit fragments need enough boundary data for safe composition and projection. | Replacements could compare functions over incompatible interfaces. | Schematic example: boundary `(a,b)`, output `(o)`, profile `{constants: [0,1], obligations: []}` for one NAND output. | Report: Package E and framework match. | VerifyDW soundness; FinalMatch. |
| full mode | `downloads/canonical_proof_report.tex`, mode sections | Semantics retaining all full carrier information and obligations. | Concrete semantics. | Needed to distinguish real replacements from quotient comparisons. | Invalid quotient-to-full transfer. | `modeUse.fullLiftRequired = true` in fixture. | Report: `CheckModeUse0`, Package E. | Mode firewall theorem. |
| quotient mode | `downloads/canonical_proof_report.tex`, mode sections | Projected semantics that may forget bits or finite-kernel information and can be used for comparison only unless lifted. | Abstract semantics or quotient semantics. | Projection enables comparison but must not authorize construction. | Same as mode firewall failure. | Mode-firewall failing fixture. | Report: mode checker, transfer identity checks. | Projection monotonicity; transfer identity. |
| obligation | `downloads/canonical_proof_report.tex`, Package E | Ledger entry created by certain rewrites and discharged only by allowed full-mode evidence. | Proof obligation. | Rewrites can defer conditions, but they must be closed. | Open obligations could hide unsound replacements. | Fixture requires `openObligations: 0`. | Report: Package E obligation lifecycle; local `checkModeFirewall`. | VerifyDW soundness. |
| charge | Report Package E and row generation sections | Integer accounting record for size/resource equalities and telescoping. | Potential function or resource ledger. | Size decrease claims need exact accounting. | False gain or polynomial bound. | Schematic example: replacement removes one NAND gate, so the charge ledger records `old_size 6`, `new_size 5`, `gain 1`. | Report: charge validators and arithmetic-cell certificates. | Charge soundness; checker hardening. |

| route | Many package sections | Named outcome or proof path selected for a row or branch, usually by priority. | Case split branch or proof rule application. | The proof is organized by deterministic case routing. | A lower-priority or missing route could hide a counterexample. | Minimal fixture does not model route priority. | Report: `CheckRoutePriority0` and package route ledgers. | Checker hardening. |
