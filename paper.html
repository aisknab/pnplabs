Review guide

Guide to reviewing the P versus NP claim.

This guide shows how the report, conditional theorem boundary, locked NAND exact minimization route, generated package checking, final acceptance replay, and release records fit together. It does not imply external validation.

Claim boundary

The main conditional theorem.

The public theorem uses a conditional checker-acceptance statement. The report records the internal package/replay/certificate chain that is claimed to discharge that condition in the sealed artefact stack.

Public theorem

CheckPCCPackexp(GeneratePCCPack()) = accept ⇒ P = NP

The theorem statement is tied to internal checker acceptance. Reviewers can first establish what the checker accepted, what was replayed, and how those records link to the final P = NP claim.

Trust model

Generator untrusted; package checked

The generator emits the package, but the release claim is made at the level of checked artefacts: package checker, acceptance run, replay, final verdict, final certificate, release audit, release gate, and final report.

Reading order

Suggested reading order.

Begin with the public status and dependency structure, then move into line-by-line technical review.

Status and claim boundary

Confirm the theorem record, antecedent, consequent, conditional flag, checker trust model, and lack of implied external consensus.

Direct-wire framework

Track carriers, open functions, compatible supports, replacement, charge ownership, and the mode firewall.

Residual-band minimization

Follow NormalizeOrGain, HResolve, BudgetResolve, selector realization, HB closure, ZeroSlack, and PCCMin.

Locked NAND SAT embedding

Check the threshold construction, macro distinctness, trace equivalence, final-lock separation, and residual slack bound.

CheckPack and package manifest

Audit row keys, package references, import graph, no-hidden-minimization scanning, reflection, and polynomial bounds.

Final release record

Inspect checker-accepted report statements, digest ledger, sealed release identifiers, and the two independent verification modes.

Core concepts

Technical terms used throughout the report.

Residual slackΛ(C) = |C| − μ(C), the gap between a circuit and its minimum equivalent size.The descent measure is residual slack, not root circuit size.

No-hidden minimizationA scan rejecting executable uses of hidden minimization or canonical optimality symbols.It supports the claim that the checker remains explicit and polynomial.
