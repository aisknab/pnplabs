Checked Lean milestone

Axiom-free direct-wire NAND core

Complete in this milestone

Typed topological NAND programs, Boolean evaluation, ordered output wiring, size laws, equivalence laws, and representative identity, NAND, NOT, and AND examples are formalized. leanNANDDirectWireCoreFormalized = true and its axiom audit passes.

Still incomplete

leanNANDEnumeratorFormalized = false. Exact minimum and slack, compatible replacement, the global slack law, locked-NAND builder and threshold, SAT, and the root theorem also remain false or absent.

This foundational semantics milestone is not a proof of the locked-NAND reduction, SAT in P, or P = NP.

What completion must demonstrate

Pinned root build

lake build PNP now compiles the explicit library root under leanprover/lean4:v4.31.0. Its PNP.Main.rootTheoremStatus declaration is status data, not the target theorem.

Root theorem

PNP.Main.p_eq_np must be present, build successfully, and pass an axiom audit without project-specific axioms. All three root-theorem status fields are currently false.

Five project-specific axioms remain: PNP.SAT, PNP.LockedNANDThreshold, PNP.ResidualBandExactMinimization, PNP.GeneratePCCPack, and PNP.CheckPCCPackexp.

Checker boundary

Legacy acceptance is historical assertion-checker evidence

The legacy JavaScript checker stack verifies assertion-bearing records under implemented predicates. It does not formalise or prove the named mathematical propositions. Its prior activation coordinate, run registry, comparison matrix, and badge are preserved for auditability only.

  • public/pnp-verification-runs.json is a frozen historical registry.
  • public/pnp-verifier-run-comparison-matrix.json is a historical comparison snapshot.
  • public/pnp-verifier-run-matrix-summary.json is neutral historical metadata, not a current green badge.
  • public/pnp-one-command-upload.json records that activated-run intake is frozen.
